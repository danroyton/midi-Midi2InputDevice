# 前端规范 – MidiController

## 1. 概述

前端是一个 **WPF 应用程序（.NET 10）**，同时以进程内方式托管后端（单 EXE 模型）。它通过 **REST**（localhost:5000）与后端通信，并通过 **SignalR WebSocket** 接收实时事件。应用程序不存储自己的配置——所有数据均通过后端 API 加载和保存。

> **单 EXE：** 后端 Kestrel 主机在启动时通过 `BackendHostService` 自动以进程内方式启动。无需单独的后端进程。

├──────────┬───────────────────────────────────────────────┤

- **左侧导航**：在五个主要区域之间切换。

### 2.2 视图：设备

**用途：** 列出物理 MIDI 设备并管理虚拟端口。

```
┌─ 物理设备 ──────────────────────────────────────────────┐
│ [刷新]                                                  │
│                                                         │
│  ► KORG nanoKONTROL2   [打开]                          │
│  ► Arturia KeyStep      [打开]                          │
└─────────────────────────────────────────────────────────┘
┌─ 虚拟端口 ──────────────────────────────────────────────┐
│  [+ 新建]                                               │
│                                                         │
│  MidiCtrl-A  → KORG nanoKONTROL2  [移除]               │
│  MidiCtrl-B  → KORG nanoKONTROL2  [移除]               │
└─────────────────────────────────────────────────────────┘
```

**操作：**
- `GET /api/v1/devices` → 刷新设备列表。
- `POST /api/v1/devices/virtual` → 创建新虚拟端口（对话框：名称 + 源设备）。
- `DELETE /api/v1/devices/virtual/{id}` → 移除端口。

### 2.3 视图：日志（MIDI 原始数据）

**用途：** 实时输出设备的 MIDI 原始数据。

```
┌─ 设备: [KORG nanoKONTROL2 ▼]  [开始] [停止] [清空] ────┐
│ 时间戳       │ 类型          │ Ch │ Data1 │ Data2        │
│──────────────┼───────────────┼────┼───────┼──────────────│
│ 00:00:01.023 │ ControlChange │  1 │    16 │          64  │
│ 00:00:01.187 │ NoteOn        │  1 │    60 │         100  │
│ ...          │               │    │       │              │
└──────────────────────────────────────────────────────────┘
```

**实现：**
- SignalR 连接到 `/hubs/midilog`。
- 客户端订阅所选设备的事件。
- 最大缓冲区：1,000 行（最旧的行被丢弃）。
- 右键单击某行 → **"为此事件创建触发器"**（打开预填充的触发器编辑器）。

### 2.4 视图：映射（触发器配置）

**用途：** 管理配置文件的映射规则（触发器）。

#### 触发器列表

```
┌─ 触发器  [+ 新建] ──────────────────────────────────────┐
│  # │ 设备               │ 事件        │ 动作        │ 编辑 │
│────┼────────────────────┼─────────────┼─────────────┼──────│
│  1 │ nanoKONTROL2 Ch1   │ CC #16      │ 音量↑ / 音量↓ │ ✎ 🗑│
│  2 │ KeyStep Ch1        │ NoteOn #60  │ 空格 ×3     │ ✎ 🗑 │
└─────────────────────────────────────────────────────────┘
```

#### 触发器编辑器（对话框 / 侧面板）

```
┌─ 编辑触发器 ────────────────────────────────────────────┐
│ [▶ 录制]  [■ 停止]  （从上次事件预填充）                │
│                                                         │
│ 设备：      [KORG nanoKONTROL2 ▼]                       │
│ 事件类型：  [ControlChange ▼]  通道: [1]  Data1: [16]   │
│ 匹配：      [Variable ▼]  值: [0]                       │
│                                                         │
│ ── 步骤 0：全局预赋值 ───────────────────────────────── │
│ [+ 赋值]                                                │
│   变量 [B▼] ← [Fixed▼] 值 [1]                    [✕]   │
│                                                         │
│ ── 条件块（AND 链接）────────────────────────────────── │
│ [+ 块]                                                  │
│ 块 1  模板:[only-when-active▼]              [✕ 块]     │
│ 块 2  模板:[-- 内联 --▼]                    [✕ 块]     │
│   [+ 条件]                                              │
│   左 [MidiData2▼] 运算符 [>=▼] 右[Fixed▼][10]   [✕]   │
│   ELSE（若块 2 失败）: [▼ 配置…]                       │
│                                                         │
│ ── 动作（顺序执行）──────────────────────────────────── │
│ [+ 动作]                                                │
│ 动作 1  模板:[-- 内联 --▼]  [↑][↓][✕]                  │
│   按键: [VolumeUp]  [+ 键]                              │
│   X (重复): ● DD2Positive  ○ DD2Negative  ○ 固定[_]    │
│   Y (按键时长): ● 固定 [0]                              │
│   Z (暂停): ● 固定 [0]                                  │
│                                                         │
│ ── 全局后赋值（所有动作之后）────────────────────────── │
│ [+ 赋值]                                                │
│   变量 [B▼] ← [Fixed▼] 值 [0]                    [✕]   │
│                                                         │
│         [取消]  [另存为模板…]  [保存]                   │
└─────────────────────────────────────────────────────────┘
```

**字段说明：**

| 区域 | 字段 | 类型 | 说明 |
|---|---|---|---|
| 标头 | 录制 | 按钮 | 启动 MIDI 录制；点击某行时预填充设备/类型/通道/Data1 |
| 标头 | 设备 / 事件类型 / 通道 / Data1 | 下拉框 / 微调框 | MIDI 事件匹配过滤器 |
| 标头 | 匹配（字段 4） | 下拉框 | `Variable` \| `Data2` \| `DeltaData2` \| `DD2Positive` \| `DD2Negative` |
| 标头 | 值 | 微调框 | 匹配目标值（`Variable` 时隐藏） |
| 步骤 0 | 全局预赋值 | 列表 | 在条件块**之前**设置变量 |
| 条件块 | 块列表 | 动态 | 1..n；通过拖放排序 |
| 条件块 | 条件 | 列表 | 左 / 运算符 / 右；来源：所有变量 A–Z、MidiData1/2、DeltaData2、DD2Positive、DD2Negative、Fixed |
| 动作 | 动作列表 | 动态 | 1..n；通过 ↑/↓ 排序 |
| 动作 | 模板 | 下拉框 | ActionBlock 模板或内联 |
| 动作 | 按键 | 标签编辑器 | 修饰键 + 主键；空 = 仅状态更改 |
| 动作 | X（`Repeat`） | 单选 + 微调框 | MidiData1/2、DeltaData2、DD2Positive、DD2Negative、变量 A–Z、Fixed |
| 动作 | Y（`KeyDuration`） | 单选 + 微调框 | 同 X |
| 动作 | Z（`Pause`） | 单选 + 微调框 | 同 X |
| 页脚 | 全局后赋值 | 列表 | 所有动作完成后执行一次 |

- **复制（📋）：** 以新名称创建副本。
- 所有操作通过 `GET/POST/PUT/DELETE /api/v1/templates`。

## 3. API 通信

### 3.1 HTTP 客户端

- `HttpClient`，`BaseAddress = http://localhost:5173`。
- 重试策略（Polly）：3 次尝试，指数退避。
- 超时：每次请求 5 秒。
- 每 5 秒通过 `GET /api/v1/status` 定期检查连接状态。

### 3.2 SignalR

- `HubConnectionBuilder`，自动重连。
- 连接断开时：状态指示器变红；后台重新连接。
