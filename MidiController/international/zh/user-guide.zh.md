## 1. 系统要求

| 要求 | 详情 |
|---|---|
| **操作系统** | Windows 10 或 Windows 11（64 位） |
| **.NET 运行时** | 不需要——已内置于 EXE 中 |
| **MIDI 硬件** | 至少一个物理 MIDI 设备（USB 或通过驱动程序的 DIN） |
| **内存** | 64 MB RAM（推荐：128 MB） |
| **磁盘空间** | 约 120 MB |

## 2. 安装与首次启动

### 2.1 下载

1. 从 GitHub 的 **Releases** 下载最新版本：
   https://github.com/danroyton/midi-Midi2InputDevice/releases/latest

2. 将 ZIP 压缩包解压到任意位置，例如：
   `C:\Program Files\MidiController\`

3. 解压后的文件夹至少包含：
   ```
   MidiController.Frontend.exe
   appsettings.json
   appsettings.backend.json
   ```

### 2.2 首次启动

1. 双击 **`MidiController.Frontend.exe`** 启动。
   首次启动时，Windows Defender SmartScreen 可能显示警告 → 点击**"仍要运行"**。

2. 应用程序打开主窗口，并自动在后台启动后端服务。

3. 右上角的**状态图标**显示连接状态：
   - 🟢 绿色 = 后端已启动，配置文件已激活
   - 🟠 橙色 = 后端已启动，门控已锁定（首次启动的默认状态）
   - 🔴 红色 = 后端尚未就绪（稍等片刻）

4. **首次启动**时没有活动的配置文件。请先创建一个配置文件（→ 第 5 节）。

### 2.3 配置文件

配置文件和模板数据存储在 `%APPDATA%\MidiController\`
（通常为：`C:\Users\<用户名>\AppData\Roaming\MidiController\`）。

EXE 旁边的 `appsettings.backend.json` 文件控制后端设置
（端口、数据路径）。通常不需要更改。

## 4. 连接 MIDI 设备

### 4.1 打开设备视图

导航 → **设备**

### 4.2 打开设备

1. 列表显示操作系统识别的所有 MIDI 输入设备。
2. 点击所需设备旁的 **[打开]**。
3. 状态变为**"已连接"**。

> **提示：** 如果设备未出现，请检查 MIDI 驱动程序并点击 **[刷新]** 重新加载列表。

### 4.3 断开设备

点击已连接设备旁的 **[关闭]**。

## 5. 管理配置文件

**配置文件**将特定使用场景的所有触发器配置归为一组
（例如"游戏"、"视频编辑"、"直播控制"）。

### 5.1 创建配置文件

1. 在配置文件下拉框中选择**"+ 新建配置文件…"**。
2. 输入名称并确认。
3. 新配置文件出现在下拉框中。

### 5.2 激活配置文件

1. 在下拉框中选择配置文件。
2. 点击 **[激活]** → 门控设为 A=0，触发器开始被评估。

### 5.3 删除配置文件

仅在没有活动配置文件时可用：
在下拉框中选择配置文件 → 点击 **[删除配置文件]**。

## 6. 创建和编辑触发器

**触发器**定义：*哪个 MIDI 事件*触发*哪个按键动作*。

### 6.1 创建新触发器

1. 导航 → **映射**。
2. 点击 **[+ 新建]**。
3. 触发器编辑器打开。

### 6.2 定义 MIDI 来源

| 字段 | 含义 |
|---|---|
| **设备** | 哪个 MIDI 设备触发？ |
| **事件类型** | `NoteOn`、`NoteOff`、`ControlChange`、`ProgramChange`、… |
| **通道** | MIDI 通道 1–16 |
| **Data1** | 音符编号或 CC 编号（0–127） |
| **匹配** | `Variable`（始终触发）、`Data2`（固定值）、`DD2Positive` / `DD2Negative`（用于旋转编码器） |

> **提示：** 使用 **[▶ 录制]** 在设备上直接触发 MIDI 事件——编辑器会自动填写设备、类型、通道和 Data1。

### 6.3 全局预赋值

在评估所有条件块**之前**需要设置的变量。
典型用途：设置锁定变量，防止另一个触发器同时执行。

### 6.4 条件块

条件块决定动作**是否**被执行。

- 块之间：**AND** 逻辑（所有块必须满足）。
- 块内：**OR** 逻辑（至少一个条件必须满足）。
- 每个块可以引用**模板**或内联定义自己的条件。
- 每个块有一个 **ELSE 分支**，允许在失败时执行替代动作。

条件格式：`[左值] [运算符] [右值]`

| 来源（左/右） | 说明 |
|---|---|
| `Fixed` | 固定数值 |
| `MidiData1` / `MidiData2` | 触发事件的原始 MIDI 数据字节 |
| `DeltaData2` | 与上次事件的 Data2 差值 |
| `DD2Positive` / `DD2Negative` | 正/负变化量的绝对值 |
| `A`…`Z` | 状态变量 |

运算符：`=`、`≠`、`<`、`≤`、`>`、`≥`

### 6.5 动作

动作向 Windows 发送按键。

| 字段 | 说明 |
|---|---|
| **按键** | 修饰键（Ctrl、Alt、Shift、Win）+ 主键 |
| **X（Repeat）** | 按键按下多少次？来源：`Fixed`、`DD2Positive`、变量 A–Z、… |
| **Y（KeyDuration）** | 按键保持多长时间（ms）？ |
| **Z（Pause）** | 按键后暂停（ms） |
| **状态赋值** | 此次按键后设置变量 |

> **旋转编码器提示：** 对于旋转编码器（如音量），创建两个动作：
> - 动作 1：`VolumeUp`，X = `DD2Positive` → 向右旋转增大音量
> - 动作 2：`VolumeDown`，X = `DD2Negative` → 向左旋转减小音量

### 6.6 全局后赋值

所有动作**之后**设置的变量（执行一次，与条件块无关）。
典型用途：释放锁定变量。

### 6.7 保存触发器

点击 **[保存]**。触发器出现在映射视图的触发器列表中。

## 7. 了解映射引擎

### 7.1 激活门控（变量 A）

仅当 **A = 0**（活动）时处理才会运行。

| 值 | 状态 | 图标 |
|---|---|---|
| `0` | 活动——触发器被评估 | 🟢 |
| `1` | 暂停——接收事件但不评估 | 🟡 |
| `2` | 锁定——启动时的默认状态；不接收 | 🟠 |

### 7.2 状态变量 A–Z

- 26 个状态变量（A 至 Z），值范围：−127 至 +127
- 可在触发器动作中读取和写入
- 保留变量（不用于一般用途）：

| 变量 | 别名 | 含义 |
|---|---|---|
| A | `ActiveListen` | 激活门控（0/1/2） |
| X | `Repeat` | 重复次数的默认值 |
| Y | `KeyDuration` | 按键保持时长的默认值（ms） |
| Z | `Pause` | 按键后暂停的默认值（ms） |

### 7.3 差值追踪

每次 MIDI 事件都会自动计算：

| 值 | 说明 |
|---|---|
| `DeltaData2` | 差值：当前 Data2 − 上次事件的 Data2 |
| `DD2Positive` | 当 `DeltaData2 > 0` 时的值；否则为 0 |
| `DD2Negative` | 当 `DeltaData2 < 0` 时的绝对值；否则为 0 |

非常适合相对旋转编码器（无限旋钮）。

## 8. 键盘测试

导航 → **测试**

无需 MIDI 硬件即可**直接测试按键**：

1. 启用修饰键复选框（Ctrl、Alt、Shift、Win）
2. 输入主键（如 `T` 或 `VolumeUp`）
3. 设置 Y（时长）和 Z（暂停）
4. 点击 **[▶ 发送]**

上次发送的按键显示在**"上次动作"**区域。

## 9. MIDI 日志

导航 → **日志**

MIDI 日志实时显示所有接收到的 MIDI 事件：

- **设备过滤器**：仅显示特定设备的事件
- 最多 1,000 行；旧条目被丢弃
- **[清空]**：清除列表
- **右键单击**某行 → "为此事件创建触发器"
  （打开带预填充字段的触发器编辑器）

## 10. 系统托盘

应用程序启动后，托盘图标出现在 Windows 任务栏右下角。

**颜色：**

| 颜色 | 含义 |
|---|---|
| 🟢 绿色 | 已连接 + 活动（A=0） |
| 🟡 黄色 | 已连接 + 暂停（A=1） |
| 🟠 橙色 | 已连接 + 锁定（A=2） |
| 🔴 红色 | 后端不可访问 |

**闪烁：** 每次接收到 MIDI 事件时图标短暂闪烁（仅在实际活动时）。

**右键菜单：**
- `激活 (A=0)` / `暂停 (A=1)` / `锁定 (A=2)` → 直接切换门控
- `显示窗口` → 打开主窗口
- `退出` → 关闭应用程序（后端将被停止）

**最小化窗口：** 主窗口可以关闭——应用程序继续在托盘中运行。
通过双击托盘图标或在右键菜单中选择"显示窗口"重新打开。

## 11. 常见问题（FAQ）

**Q：我的 MIDI 设备未出现在设备列表中。**
A：确保 MIDI 驱动程序已安装且设备出现在设备管理器中。然后点击设备视图中的"刷新"。

**Q：键盘输入未发送到我的游戏/应用程序。**
A：目标窗口必须获得焦点。MidiController 本身不能在前台。如有疑问，将目标应用程序置于前台，然后从 MIDI 设备触发。

**Q：启动后门控已锁定（A=2）——为什么？**
A：这是首次启动的默认行为。在状态选项卡中或通过托盘右键菜单点击"激活"以将 A 设为 0。

**Q：我的配置文件存储在哪里？我能备份吗？**
A：配置文件以 JSON 文件形式存储在 `%APPDATA%\MidiController\profiles\` 下。只需复制此文件夹即可。

**Q：我能同时使用多个 MIDI 设备吗？**
A：可以。在设备视图中打开多个设备。每个触发器可以分配给特定设备。

**Q：如何更改后端运行的端口？**
A：在 `appsettings.backend.json`（EXE 旁边）中调整 `Urls` 值，例如：
`"Urls": "http://localhost:5001"`。同时在 `appsettings.json` 的 `Backend.BaseUrl` 中设置相同的值。

**Q：我能将应用程序作为 Windows 服务运行吗？**
A：当前版本不完全支持。`SendInput` 在没有会话转发的情况下无法在服务模式（Session 0）中工作。计划在 v1.0 中实现。

## 12. 故障排除

### 后端未启动（状态持续为红色）

1. 检查端口 5000 是否已被占用：
   ```powershell
   netstat -ano | findstr :5000
   ```
2. 如果被占用，在 `appsettings.backend.json` 中更改端口。
3. 检查事件日志：Windows 事件查看器 → 应用程序日志。
4. 以管理员权限启动 `MidiController.Frontend.exe`（仅用于测试）。

### MIDI 事件已收到，但没有键盘输入发送

1. 在**状态选项卡**中确保 A=0（绿色）。
2. 检查触发器定义：设备、通道、Data1 必须与 MIDI 日志完全匹配。
3. 在 **MIDI 日志**中验证事件确实被接收。
4. 检查目标窗口是否获得焦点。

### 触发器触发但按键不正确

1. 在**测试选项卡**中直接发送所需按键并在目标窗口中验证。
2. 确保没有无意中设置修饰键。
3. 对于媒体键（`VolumeUp`、`MediaPlayPause` 等），确保目标程序处理这些特殊键。

### CPU 占用率高

应用程序空闲时 CPU 使用率应低于 1%。大量 MIDI 事件时负载高属正常现象。
在无 MIDI 活动时持续高负载，请在 [GitHub](https://github.com/danroyton/midi-Midi2InputDevice/issues) 上提交 Issue。
