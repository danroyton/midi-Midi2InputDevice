# ADR-003: 多消费者访问的虚拟 MIDI 端口

| 字段 | 值 |
|---|---|
| 状态 | **已接受** |
| 日期 | 2025-05-30 |
| 决策者 | 开发团队 |

## 背景

Windows MIDI 驱动程序（WinMM）默认每个物理 MIDI 输入只允许**一个同时消费者**。当多个应用程序（如 DAW + MidiController）想同时访问同一设备时，第二次 `midiInOpen` 调用将失败。

## 选项

### 选项 A：loopMIDI（Tobias Erichsen）
- 免费的 Windows 虚拟 MIDI 环回端口驱动程序。
- 在 MIDI 社区中非常广泛使用。
- 可通过 COM 自动化（`LoopMIDI.COM`）或直接通过 WinMM API 访问。
- **优点：** 稳定，延迟低，社区信任度高。
- **缺点：** 外部依赖，需要单独安装。
- **缺点：** 无公开文档化的 .NET SDK。

### 选项 B：Windows MIDI Services（Windows 11 24H2 起）
- 无需虚拟驱动程序的原生多客户端支持。
- **优点：** 不需要外部驱动程序。
- **缺点：** 仅支持 Windows 11 24H2+；无法覆盖广泛的用户基础。

### 选项 C：后端中的自定义 MIDI 调度
- 后端独占打开物理设备并在内部分发事件。
- 其他应用程序通过虚拟端口（通过 loopMIDI 或 MIDI-Yoke 创建）接收事件。
- **优点：** 最大控制权；后端是物理端口的唯一消费者。
- **缺点：** 后端未运行时，其他应用无法访问该设备。

## 决策

**选项 C**（自定义调度）结合**选项 A**（loopMIDI 用于虚拟输出端口）。

后端独占打开物理设备。事件在内部转发，并可选择镜像到 loopMIDI 端口，以便其他应用程序可以订阅虚拟端口。

## 理由

- 选项 C 对于核心功能不依赖于 loopMIDI；loopMIDI 仅用于可选的镜像功能。
- 后端作为物理端口的唯一消费者，可实现最大控制权和最低延迟。
- loopMIDI 是 Windows 下虚拟 MIDI 端口的事实标准；安装可以在设置中自动化。

## 影响

- 后端设置会检查是否已安装 loopMIDI；若未安装则显示警告。
- `IVirtualMidiPort` 抽象层允许日后切换到 Windows MIDI Services。
- 文档必须说明 loopMIDI 的安装方式。
- 后端未运行时，其他应用程序**无法**访问物理端口。此行为须在文档中明确说明。
